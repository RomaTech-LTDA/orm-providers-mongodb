@romatech/orm-providers-mongodb

GitHub npm

MongoDB (NoSQL) provider for @romatech/orm. Same familiar ORM API — entities map to documents, tables map to collections. Supports transactions with replica sets.

Installation

npm install @romatech/orm @romatech/orm-providers-mongodb reflect-metadata

Quick Start

import 'reflect-metadata';
import { DbContext, DbContextOptions, Entity, PrimaryKey, Column } from '@romatech/orm';
import { MongoDbProvider } from '@romatech/orm-providers-mongodb';

@Entity('users')
class User {
    @PrimaryKey() id!: string;
    @Column() name!: string;
    @Column() email!: string;
}

class AppDbContext extends DbContext {
    users = this.set(User);

    constructor() {
        super(
            new DbContextOptions().useProvider(
                new MongoDbProvider({
                    uri: 'mongodb://localhost:27017',
                    database: 'myapp'
                })
            )
        );
    }
}

const db = new AppDbContext();
db.users.add({ id: 'abc123', name: 'Alice', email: 'alice@example.com' });
await db.saveChanges();

Features

Full CRUD

add, addRange, update, remove, removeRange, find, getAll — documents stored in collections.

Transactions

Multi-document transactions with beginTransaction, commit, rollback (requires replica set).

QueryBuilder

Ordering via MongoDB sort(), skip/take pagination, and client-side predicate evaluation.

Schema-less

No DDL migrations needed — MongoDB is schema-free. Migration history still tracked.

Scaffold

Infer entity types from sample documents and generate TypeScript files.

MongoDB 4.0+

Requires Node.js ≥ 18. Uses the mongodb package (installed automatically).
